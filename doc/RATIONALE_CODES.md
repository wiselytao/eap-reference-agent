# Rationale Codes

This file documents rationale codes emitted in `trace.step_plans[].rationale_codes` to make each step's plan auditable.

## Step Selection
- `R_STEP1_LLM_RELEVANCE_SOFT`: Step 1 uses LLM to decide relevance with a recall-friendly rule (include uncertain tools).
- `R_STEP1_RELEVANCE_FALLBACK`: LLM unavailable; Step 1 uses all candidate tools.
- `R_STEP1_RELEVANCE_EMPTY_STOP`: LLM marked no tools as relevant; Step 1 stops and returns a no-evidence response.
- `R_GAP_LLM_SELECTION`: Tools selected by LLM based on missing items/fields and tool summaries.
- `R_GAP_SUMMARY_MATCH`: Tools selected by heuristic matching missing items/fields against tool summaries.
- `R_GAP_UNUSED_FALLBACK`: No clear match; select tools not used yet (or all candidates if none remain).
- `R_GAP_ALL_CANDIDATES`: Gap selection fell back to all candidate tools.
- `R_GAP_NONE`: No gaps detected; no further tool selection.

## Query Construction
- `R_MULTI_QUERY_LLM`: Up to three follow-up questions generated by LLM for the step.
- `R_MULTI_QUERY_FALLBACK`: LLM unavailable/failed; reuse the current query as the only question.

## Stop Reasons (Evaluations)
- `STOP_MAX_STEPS`: Reached maximum step budget.
- `STOP_COVERAGE_COMPLETE`: Answer blueprint coverage is complete.
- `STOP_BINDINGS_MET`: All required fields have been met.
- `STOP_EVIDENCE_MIN_MET`: Evidence minimum reached with coverage complete.
- `STOP_STEP_BUDGET`: Step budget exhausted.
- `STOP_NO_PROGRESS`: No progress detected; stop to avoid wasting steps.

## Final Status Reasons
- `DOWNGRADE_NEEDS_MORE_STEPS`: Evaluation indicates more steps were needed.
- `DOWNGRADE_EVIDENCE_CONTRACT`: Evidence contract violated for required tools.
- `DOWNGRADE_NO_EVIDENCE`: No evidence available while citations required.
- `DOWNGRADE_EMPTY`: Final status is empty without evidence.
